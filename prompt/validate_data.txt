请验证以下目录数据是否符合规范，并进行必要的修正。

---输入数据---
{structured_data}
---输入结束---

验证检查项：

1. **字段完整性**
   - 每个条目必须包含：title, page, level
   - 不允许有额外的未定义字段

2. **数据类型检查**
   - title: 必须是字符串，长度 1-500
   - page: 必须是整数，范围 -1 或 1-9999
   - level: 必须是整数，范围 1-5

3. **逻辑一致性**
   - 页码应该大致递增（允许小范围波动）
   - 层级变化应该连续（不能从 level 1 直接跳到 level 4）
   - 同一层级的条目不应该有完全相同的标题

4. **常见错误修正**
   
   a) **页码错误**
      - 页码 = 0 → 修正为 1
      - 页码 > 9999 → 标记为异常
      - 页码小于前一条 → 标记为疑似错误
   
   b) **层级错误**
      - level < 1 → 修正为 1
      - level > 5 → 修正为 5
      - 层级跳跃过大（如 1→4）→ 标记为疑似错误
   
   c) **标题清理**
      - 去除多余的空格
      - 去除标题末尾的点号或破折号
      - 统一引号格式（全角→半角）

5. **输出格式**

如果数据完全正确：
```json
{
  "status": "valid",
  "data": [...],
  "warnings": []
}
```

如果发现问题但已修正：
```json
{
  "status": "valid_with_fixes",
  "data": [...],
  "warnings": [
    "第3个条目：页码从 15 跳到 5，已保留原值但需人工确认",
    "第7个条目：层级从 2 跳到 5，已修正为 3"
  ]
}
```

如果有严重错误无法自动修正：
```json
{
  "status": "invalid",
  "data": [...],
  "errors": [
    "第2个条目：标题为空",
    "第5个条目：页码不是数字"
  ]
}
```

验证示例：

输入：
```json
[
  {"title": "第一章  ", "page": 1, "level": 1},
  {"title": "1.1 背景", "page": 3, "level": 2},
  {"title": "1.1.1 现状", "page": 0, "level": 3}
]
```

输出：
```json
{
  "status": "valid_with_fixes",
  "data": [
    {"title": "第一章", "page": 1, "level": 1},
    {"title": "1.1 背景", "page": 3, "level": 2},
    {"title": "1.1.1 现状", "page": 1, "level": 3}
  ],
  "warnings": [
    "第1个条目：已去除标题末尾空格",
    "第3个条目：页码从 0 修正为 1"
  ]
}
```
